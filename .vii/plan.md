### Architecture Decision: Content Layer

Use **static TypeScript data files** in `src/data/` — no CMS, no database, free, instant. Three files:

```
src/data/projects.ts    — project cards with title, description, stack, link, image
src/data/workex.ts      — work experience entries with company, role, dates, bullets
src/data/blog.ts        — blog post stubs or MDX front matter index
```

This keeps Next.js as a pure static site (no server needed), deployable to Vercel free tier.

### Architecture Decision: Chat Widget

Two options:
- **Option A — Real AI:** Add `/api/chat` route that calls Anthropic Claude API with a system prompt describing Vibhanshu. Requires `ANTHROPIC_API_KEY` env var. Cost: ~$0 at low traffic.
- **Option B — Scripted bot:** Pre-written Q&A pairs, keyword matching. Zero cost, zero latency, works offline.

Recommendation: **Option A** — it's the differentiating feature for a founder audience. A working AI chat that actually knows Vibhanshu's experience is memorable. Use the Claude Haiku model (cheapest). Add rate limiting via a simple in-memory counter or Vercel KV if needed later.

## CEO Perspective

### What a founder/investor actually needs to see to reach out

This is a conversion problem. The visitor lands, scans in ~8 seconds, and decides: "worth a reply or not?"

Three signals that drive inbound from founders/investors:
1. **Credibility**: Where has Vibhanshu worked? What has he shipped? (social proof, logos, outcomes)
2. **Thesis**: What is he building now and why? (shows thinking, vision alignment)
3. **Personality / taste**: Does his writing/design feel sharp? (founder signal: craft = judgment)

Missing from the current site: all three. The hero copy is about an AI code review SaaS product — that's actively confusing.

### Priority Stack (ruthless ranking)

| # | What | Why it drives the metric |
|---|---|---|
| 1 | Real hero copy — who Vibhanshu is, what he's building now | First 3 seconds. Visitor decides to keep reading here. |
| 2 | Clear contact CTA above the fold | Remove friction. Interested parties must be able to reach Vibhanshu in one click. |
| 3 | Work experience with real companies | Credibility anchor. Investors pattern-match on logos. |
| 4 | 2-3 flagship projects with outcomes | Proof of shipping. Links to live products > screenshots. |
| 5 | Blog / writing (even 1-2 posts) | Signals how Vibhanshu thinks. Founders who write attract founders who read. |
| 6 | AI chat widget | Memorable differentiator — but only if it actually works. A broken chat widget is worse than none. |

### Contact Strategy

The brief says success = "3 inbound messages." Make contact trivially easy:
- Email link in hero CTA (mailto: or Calendly)
- Email + LinkedIn in footer
- "Get in touch" button in header nav

Do NOT make the contact form the only path — founders don't fill out forms. Direct email is highest-conversion.

### GO/NO-GO

**GO.** The brief is valid, the stack is sound, the constraint (free tools only) is met by the static + Vercel approach. No blocking issues.

The riskiest assumption from the brief — "qualified people will reach out" — is not de-risked by the site alone. The site makes outreach possible; Vibhanshu still needs to drive traffic (LinkedIn posts, Twitter threads, direct sharing). Build the site, then actively share it.

### What to cut if time is short

If bandwidth is limited, ship in this order and stop when good enough:
1. Minimum viable: Hero (real copy) + WorkEx (real data) + Contact CTA → deploy
2. Good: + Projects + AboutMe
3. Complete: + Blog + Chat widget

## Design Perspective

### Current Visual Problems

1. **No personal identity** — the site looks like an out-of-the-box Mantine template. Nothing signals "this is Vibhanshu."
2. **Dot pattern hero** — overused on SaaS landing pages. Reads as generic, not personal.
3. **Placeholder imagery** — Croatia beach photos in a portfolio card is jarring and breaks trust.
4. **Typography is flat** — all the same weight/size rhythm, no clear hierarchy between sections.
5. **Color: default Mantine blue** — every Mantine template uses this. Pick one accent color that's yours.
6. **Header logo: 3D cube icon** — no connection to Vibhanshu. Needs either initials ("VS"), a wordmark, or a meaningful icon.
7. **HeroText CTAs: "Book a demo" / "Purchase a license"** — SaaS copy on a personal portfolio is actively wrong.
8. **Spacing is tight** — sections stack with no breathing room. The page feels rushed.

### Design Direction: Sharp & Personal

Avoid: purple gradients, rounded-2xl-everything, glassmorphism, "AI aesthetic."

Go for: opinionated typography, generous whitespace, one strong accent, minimal but intentional visual moments.

Reference aesthetic: personal sites that feel like the person — think paco.me, rauno.me, leerob.io. Clean, confident, typographically led.

### Token System

```css
/* Recommended design tokens — apply via Mantine theme or globals.css */

--color-bg:       #0a0a0a      /* near-black — dark base */
--color-surface:  #111111      /* card/section backgrounds */
--color-border:   #222222      /* subtle separators */
--color-text-1:   #f5f5f5      /* primary text */
--color-text-2:   #a0a0a0      /* secondary/dimmed */
--color-accent:   #e8ff00      /* electric yellow — one strong accent, used sparingly */

/* Typography */
--font-display:   'Geist', system-ui, sans-serif   /* bold headlines */
--font-body:      'Geist', system-ui, sans-serif   /* clean body */

/* Spacing */
--section-gap:    120px   /* between major sections */
--content-width:  960px   /* max content width */
```

Note: Geist is available via `next/font/google` or install `geist` npm package. Alternatively use system-ui + weight contrast to differentiate headlines.

## Design Perspective — Blog Tile Grid Redesign

### User flow
1. User lands on blog panel (sidebar tab "blog.md" or `/blog` route — both use `BlogListPanel`)
2. Sees a header bar (`// blog.md`) then a scrollable 3-column tile grid of all posts
3. Hovers a tile — border accent + subtle bg shift
4. Clicks tile — navigates to post reading pane (existing `onSelectPost` callback, unchanged)
5. If > PAGE_SIZE posts: prev/next pagination appears below the grid, scrolls into view
6. Branch — 0 posts: existing "Coming Soon" empty state (unchanged)
7. Branch — mobile (≤640px): 1-column stack, tiles flow vertically, scroll works
8. Branch — tablet (640–1024px): 2-column grid
